Section 4.1 Displaying Data
Raw data on its own is hard to understand. Displaying data in charts and diagrams makes patterns and comparisons easy to see. This section covers the main ways to display data: pictograms, stem-and-leaf diagrams, bar charts, pie charts, and frequency diagrams.
W
Write This Down — Pictograms

A pictogram uses a repeated symbol to represent data. Each symbol stands for a set number of items.

Always include a key (e.g. "each circle = 4 pizzas"). Half symbols represent half the value.

Limitation: showing fractions of symbols can only give approximate values.

W
Write This Down — Stem-and-Leaf Diagrams

A stem-and-leaf diagram splits each number into a stem (the leading digit) and a leaf (the last digit).

Step 1: Choose the stem (usually tens digit).

Step 2: Write each leaf next to its stem, in the order the data appears.

Step 3: Rewrite in order (smallest to largest leaf in each row).

Step 4: Include a key (e.g. "1 | 4 means 14").

Example: Ages: 25, 17, 32, 21, 43, 28

Stem| Leaf
1| 7
2| 1 5 8
3| 2
4| 3
Key: 1 | 7 means 17

From an ordered stem-and-leaf diagram you can easily find the mode, median, and range.

W
Write This Down — Back-to-Back Stem Plots

A back-to-back stem-and-leaf diagram compares two data sets. The stems go in the middle. One set's leaves go left, the other's go right.

Each side needs its own key. This makes it easy to compare distributions (e.g. coached vs. not coached students).

T
Study Tip — Always Put Leaves in Order

The exam wants an ordered stem-and-leaf diagram. If you write the leaves randomly, you lose marks. Write the raw version first, then rewrite in order. Don't forget the key!

  • I can draw and read a pictogram with a key
  • I can create an ordered stem-and-leaf diagram with a key
  • I can draw a back-to-back stem-and-leaf diagram to compare two data sets
Section 4.1 (cont.) Bar Charts, Pie Charts & Frequency Diagrams
Bar charts, pie charts, and frequency diagrams are the most common ways to show data in exams. You need to be able to both draw them and read information from them.
W
Write This Down — Frequency Diagrams / Bar Charts

A frequency diagram (bar chart) shows how often each value occurs.

Step 1: Draw a tally chart first — go through the data marking tallies. Group in fives: |||| then cross for five.

Step 2: Count the tallies to get the frequency for each value.

Step 3: Draw the bar chart. Label both axes. Frequency goes on the vertical axis.

For grouped data (e.g. heights from 14.3 to 21.7 cm), choose class intervals like 14 ≤ s < 16, 16 ≤ s < 18, etc. The bars touch each other (no gaps) because the data is continuous.

W
Write This Down — Pie Charts

A pie chart shows proportions as slices of a circle. The whole circle = 360°.

To read a pie chart: angle ÷ 360 × total = number in that category.

To draw a pie chart: number in category ÷ total × 360 = angle for that slice.

Example: 600 people surveyed. Camping slice = 60°. How many chose camping?

1.60 ÷ 360 × 600 = 100 people
100 people chose camping
!
Don't Confuse — Pie Chart Angles Must Add to 360°

A common mistake is calculating angles that don't add up to 360°. After you work out each angle, add them all together to check. If they don't sum to 360, you made an error. Also: always use a protractor when drawing — freehand slices lose marks.

?
Why This Matters

Pie charts are worth 3–4 marks on the exam. They test your ability to work with fractions, proportions, and angles all at once. If you can do pie charts confidently, you're also practising the fraction/percentage skills needed everywhere else in maths.

T
Study Tip — The Pie Chart Formula

Memorise this: Angle = (value ÷ total) × 360. That single formula lets you draw any pie chart. Practise it with 3–4 examples until it's automatic.

  • I can draw a tally chart and frequency diagram from raw data
  • I can group continuous data into class intervals and draw a bar chart with no gaps
  • I can calculate angles for a pie chart and check they sum to 360°
  • I can read values from a pie chart using angle ÷ 360 × total

Section 4.3 Averages: Mean, Median, Mode & Range
An "average" is a single number that represents a whole data set. There are three types of average and you need to know all of them, plus the range which measures spread.
W
Write This Down — The Three Averages + Range
AverageHow to Find ItWhen to Use
MeanAdd all values ÷ number of valuesMost common. Uses all the data.
MedianPut in order, pick the middle valueNot affected by extreme values (outliers).
ModeThe value that appears most oftenUseful for categories (e.g. most popular colour).
RangeLargest value − smallest valueShows how spread out the data is.
W
Write This Down — Worked Example

Data: 5, 4, 10, 3, 3, 4, 7, 4, 6, 5

Order:3, 3, 4, 4, 4, 5, 5, 6, 7, 10
Mean:(3+3+4+4+4+5+5+6+7+10) ÷ 10 = 51 ÷ 10 = 5.1
Median:10 values → middle is between 5th and 6th values = (4+5) ÷ 2 = 4.5
Mode:4 (appears 3 times, more than any other)
Range:10 − 3 = 7
!
Don't Confuse — Mean vs. Median with Outliers

Consider exam marks: 20, 21, 21, 22, 23, 25, 27, 27, 27, 29, 98, 98

The median (25) gives a fair picture of the typical score.

The mean (35.5) is pulled up by the two extreme scores and doesn't represent the group well.

If the exam asks "which average best represents the data?", look for outliers. If there are extreme values, the median is usually better.

!
Don't Confuse — Finding the Median

You must put the data in order first. If you pick the "middle" number from an unordered list, you'll get the wrong answer.

For an even number of values: the median is the average of the two middle values. For an odd number: it's the single middle value.

T
Study Tip — "MeMoMeR" Memory Trick

Mean = add and divide. Mode = Most common. Median = Medium (middle). Range = the Reach (biggest minus smallest).

  • I can calculate the mean of a data set
  • I can find the median by ordering data and picking the middle
  • I can identify the mode
  • I can calculate the range
  • I know when the median is a better average than the mean

Section 4.1 (cont.) Scatter Graphs & Correlation
A scatter graph (scatter diagram) plots two sets of data against each other to see if there's a relationship between them. The pattern of the points tells you about the correlation.
W
Write This Down — Types of Correlation
CorrelationWhat the Points Look LikeExample
Strong positivePoints close to an upward lineMore revision → higher marks
Weak positivePoints roughly trend upward but scatteredHeight vs. arm span
Strong negativePoints close to a downward lineHigher price → fewer sales
No correlationPoints scattered randomlyMaths score vs. time to run 800m

Positive = as one goes up, the other goes up. Negative = as one goes up, the other goes down.

W
Write This Down — Line of Best Fit

When there is positive or negative correlation, you can draw a line of best fit.

Rules for the line of best fit:

• It should follow the general trend of the points.

• Roughly equal numbers of points should be on each side of the line.

• It does NOT have to pass through the origin (0, 0).

• Use it to estimate values — e.g. "If a student scored 52 in physics, estimate their maths mark."

!
Don't Confuse — Correlation Does Not Mean Cause

Just because two things are correlated doesn't mean one causes the other. Ice cream sales and drowning rates are both high in summer — but ice cream doesn't cause drowning! The exam sometimes tests whether you understand this.

T
Study Tip — Describing Correlation on the Exam

When the exam says "describe the correlation", use two words: the strength (strong/weak) and the direction (positive/negative). Example: "strong positive correlation." If there's no pattern, say "no correlation." Don't just say "they are related."

  • I can plot a scatter graph from a table of data
  • I can describe correlation as strong/weak positive, strong/weak negative, or no correlation
  • I can draw a line of best fit and use it to estimate values
  • I understand that correlation does not prove cause

Section 4.4 Frequency Polygons
A frequency polygon is formed by joining the midpoints of the tops of the bars in a bar chart with straight lines. It's useful for comparing two data sets on the same axes.
W
Write This Down — How to Draw a Frequency Polygon

Step 1: Find the midpoint of each class interval. Midpoint = (lower boundary + upper boundary) ÷ 2.

Step 2: Plot each midpoint against its frequency.

Step 3: Join the points with straight lines.

Example: Class 50 ≤ l < 60 with frequency 4.

1.Midpoint = (50 + 60) ÷ 2 = 55
2.Plot the point (55, 4)
Repeat for all classes, then join the dots.
?
Why This Matters

Frequency polygons let you overlay two distributions on the same graph for easy comparison. The exam might give you "before and after" data (like IQ scores before and after vitamin pills) and ask you to draw two polygons on the same axes and compare.

W
Write This Down — Discrete vs. Continuous Data

Discrete data can only take certain values: number of children, shoe size, dice score.

Continuous data can take any value within a range: height, weight, time.

For continuous data, bars in frequency diagrams touch (no gaps). For discrete data, bars have gaps between them.

  • I can calculate midpoints and draw a frequency polygon
  • I know the difference between discrete and continuous data
  • I know when bar chart bars should touch vs. have gaps
Putting It All Together Key Study Strategies for Handling Data
T
The Number One Rule

Label everything. Every chart needs labelled axes, a title, and a key where relevant. On the exam, missing labels cost marks even if the chart itself is perfect. Get into the habit now: title, axes, units, key.

T
Show Your Working for Averages

For the mean, write out the full sum and the division. Even if your arithmetic is wrong at the end, you get method marks. For the median, write the ordered list — the examiner can see you know the method.

T
Check, Check, Check

Every topic in this chapter has a way to check your answer:

• Pie charts: Do all angles add to 360°?

• Mean: Is the answer between the smallest and largest values?

• Median: Did you order the data first? Is your middle value actually in the middle?

• Frequency tables: Do the frequencies add up to the total number of data points?

• Scatter graphs: Does your line of best fit have roughly equal points on each side?
